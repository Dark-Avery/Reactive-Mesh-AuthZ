# Объём реализованной системы

Ниже приведён практический ответ на вопрос «сколько кода реализовано в проекте».

Подсчёт выполняется скриптом:

```bash
python3 scripts/count_project_loc.py
```

Скрипт считает только авторский код и конфигурацию проекта и исключает:

- сгенерированные protobuf-файлы;
- временные и результирующие артефакты экспериментов;
- PDF, изображения, архивы и бинарные файлы;
- локальные материалы `deliverables/`;
- внешние исходники `envoy-src/` и содержимое `distfiles/`.

## Текущие значения

- чистое ядро реактивного IAM: **2092 строки**
- ядро реактивного IAM вместе с baseline-контуром сравнения: **2610 строк**
- ядро реактивного IAM вместе с проверочным и экспериментальным контуром: **7044 строки**
- весь авторский код и конфигурация проекта: **9992 строки**

Под «чистым ядром реактивного IAM» здесь понимаются:

- `receiver/cpp/`
- `envoy-custom/reactive_pep/`
- `idp/demo/`
- `grpc/cpp/`
- `grpc/proto/`

Под «ядром вместе с baseline-контуром» дополнительно понимается:

- `baseline/opa/`

Под «ядром вместе с проверочным и экспериментальным контуром» здесь понимаются:

- `receiver/`
- `baseline/opa/`
- `grpc/`
- `idp/`
- `envoy-custom/`
- `bench/load-client/`
- `scripts/`
- `formal/tla/`

## Что удобно говорить на защите

Если нужен один короткий ответ именно про реализованный реактивный IAM, безопасная формулировка такая:

> Чистое ядро реактивного IAM составляет около 2.1 тысячи строк. Если считать baseline-контур сравнения, то около 2.6 тысячи строк. Если считать весь проверочный и экспериментальный контур, то около 7 тысяч строк.

Если нужен более узкий инженерный акцент, можно дополнить:

- крупнейший файл проекта: `receiver/cpp/main.cpp`
- реактивный модуль Envoy расположен в `envoy-custom/reactive_pep/`
- формальная часть вынесена в `formal/tla/`
