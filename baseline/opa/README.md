# OPA + Envoy

`policy.rego` задаёт основной открытый контур сравнения `OPA + Envoy`:

- Envoy вызывает OPA через gRPC `ext_authz`;
- OPA запрашивает текущее состояние в `receiver /state`;
- если любой из `sub`, `sid` или `jti` уже помечен как запрещённый, новый поток не допускается;
- уже активный поток не завершается задним числом, поэтому эта архитектура сравнения отражает авторизацию именно в момент запроса.

Файл `authz_service.cpp` используется как служебный адаптер для дополнительных внутренних режимов проверки и для подключения внешних контуров сравнения.
